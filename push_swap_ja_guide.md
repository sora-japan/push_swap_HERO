# push_swap やること整理ガイド

---

## まず全体像を把握する

### このプロジェクトでやること

**2人で** `push_swap` という C プログラムを作る。  
このプログラムは「整数のリストを受け取って、ソートするための操作手順を出力する」もの。

```
$> ./push_swap 5 2 8 1 3
pb
pb
sa
pa
pa
ra
```

↑ こういう操作列を stdout に出力するだけ。実際にソートする必要はない。

---

## 使える操作（11種類）

2つのスタック `a`（最初に数値が入っている）と `b`（最初は空）を操作する。

```
スタックのイメージ（上が先頭）:

  [2]  [ ]
  [5]
  [1]
   a    b
```

| 命令 | 動作 |
|------|------|
| `sa` | a の先頭2つを入れ替え |
| `sb` | b の先頭2つを入れ替え |
| `ss` | sa と sb を同時 |
| `pa` | b の先頭を a の先頭に移す |
| `pb` | a の先頭を b の先頭に移す |
| `ra` | a を上に1つ回転（先頭が末尾へ） |
| `rb` | b を上に1つ回転 |
| `rr` | ra と rb を同時 |
| `rra` | a を下に1つ回転（末尾が先頭へ） |
| `rrb` | b を下に1つ回転 |
| `rrr` | rra と rrb を同時 |

#### コマンドライン引数の仕様

```bash
# 整数リストを渡すと操作を出力（デフォルトは --adaptive）
./push_swap 5 2 8 1 3

# 戦略を明示的に指定できる
./push_swap --simple   5 2 8 1 3   # O(n²) で強制
./push_swap --medium   5 2 8 1 3   # O(n√n) で強制
./push_swap --complex  5 2 8 1 3   # O(n log n) で強制
./push_swap --adaptive 5 2 8 1 3   # 自動選択（デフォルトと同じ）

# ベンチマーク情報を stderr に出力
./push_swap --bench 5 2 8 1 3
```

#### エラーケース（stderr に "Error\n" を出力）
- 整数でない引数（例: `one`）
- int 範囲外の数値
- 重複した数値

#### 引数なし
- 何も表示せず終了

## 4つのアルゴリズム戦略（全部実装が必須）

| フラグ | 計算量 | 役割 |
|--------|--------|------|
| `--simple` | O(n²) | 挿入ソート・選択ソート系 |
| `--medium` | O(n√n) | チャンク分割系 |
| `--complex` | O(n log n) | クイックソート・マージソート系 |
| `--adaptive` | disorder に応じて自動切り替え | デフォルト動作 |

### disorder（乱雑度）とは

0〜1 の数値。0 = すでにソート済み、1 = 完全に逆順。

計算方法：スタック内のすべての数値ペア (i, j) で `a[i] > a[j]` となる「逆転ペア」の割合。

```python
# イメージ（疑似コード）
mistakes = 0
total_pairs = 0
for i in range(len(a)):
    for j in range(i+1, len(a)):
        total_pairs += 1
        if a[i] > a[j]:
            mistakes += 1
disorder = mistakes / total_pairs
```

> **disorder は最初に1回だけ計算する（操作前）**

### adaptive の切り替え基準

| disorder の値 | 使うアルゴリズム |
|---|---|
| 0.2 未満（ほぼソート済み） | O(n²) |
| 0.2 以上 0.5 未満 | O(n√n) |
| 0.5 以上（かなり乱れている） | O(n log n) |

## ボーナス：checker プログラム

```
提出ファイル: *.h, *.c（Makefile の bonus ルールでビルド）
```

`push_swap` の出力が正しくソートできるかを検証するプログラム。

```bash
# 使い方
$> ./push_swap 3 2 1 0 | ./checker 3 2 1 0
OK

$> echo "sa\nrra\npb" | ./checker 3 2 1 0
KO
```

動作：
1. 引数としてスタック a の初期値を受け取る
2. stdin から操作列を読み込む（1行1命令）
3. 操作を実行して、a がソート済み・b が空なら `OK`、それ以外は `KO`
4. エラー時は `Error`

> ボーナスは必須パートが**完璧**（全ベンチマーク通過）でないと評価されない

## よくある疑問

**Q: 操作列の「最小」って最適解を求めないといけないの？**  
A: 「できるだけ少なく」という意味だが、数学的最適解は不要。ベンチマークの操作数上限を下回ればよい。

**Q: --adaptive がデフォルトってどういうこと？**  
A: フラグを何も指定しなかったとき、自動的に `--adaptive` と同じ挙動になる。

**Q: 4つの戦略は全部使われるの？**  
A: `--simple` `--medium` `--complex` は強制指定用。`--adaptive` が実際の運用で使われる。ただしバイナリには4つ全部組み込む必要がある。

**Q: disorder の閾値（0.2 / 0.5）は変えていいの？**  
A: 自分で決めてよい。ただし README にその根拠を書くこと。
